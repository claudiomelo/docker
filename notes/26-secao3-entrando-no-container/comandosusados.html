Entrar dentro de containers para executar comandos

Comandos utilizados

docker pull imageName (Faz um pull na imagem informada no dockerhub, similar ao git pull no git)

docker image ls (Exibe lista de images que baixei)

docker container run -it (Inicia um novo container interativamente)

docker container run exec -it (Executa comandos adicionais em um container)

docker container start -ai containerName bash (Inicia um container e entrar no terminal dele)

docker container exec -it containerName bash (Entrar no bash de um container que está em execução para fazer algo nele, ao sair digitanto exit o container não é desligado)

Criando os 3 containers

docker container run -it --name nginx nginx bash (Iniciar um container interativo e entra no bash do container para que possamos rodas comandos dentro do container criado

docker container run -it --name ubuntu ubuntu (Iniciando um container com ubuntu, como ubuntu tem o bash como padrão não preciso espeficicar ele no fim do comando, posso então instalar o que quiser na minha imagem do ubuntu, e quando der "exit" e ele parar o container, se eu dar um start nele ele vai ter o que instalei
