Criar 3 containers, apache (hpptd), nginx, mysql, e pegar informações sobre os containers

Comandos utilizados

docker container run -d --name containerName containerImage (cria um container baseado em uma imagem como nome informado e desatachado rodar em background)

docker container start containerNameOrId (Inicia um container parado)

docker container top (Exibe lista de processos em um container, é possivel especificar o container que deseja)

docker container inspect (Detalhes de configuração de um container)

docker container stats (Estatisticas de performance dos containers)

docker container ls (Lista os containers criados )

docker container stop containerNameOrID (para um container)

docker container ls -a (Lista os containers menos os que foram parados, * não removidos)

docker container rm containerNameOrID (remover um container)

Criando os 3 containers

docker container run -d --name nginx nginx (Iniciar um container -d=deteached --name dar nome ao container, nginx= imagem do container), caso o container ja exista esse comando da erro, e basta executar: docker run -d nginx

docker container run -d --name httpd httpd (Iniciar um container -d=deteached --name dar nome ao container, httpd= imagem do container), caso o container ja exista esse comando da erro, e basta executar: docker run -d httpd

docker container run -d --name mysql -e MYSQL_RANDOM_ROOT_PASSWORD=true mysql (Iniciar um container -d=deteached --name dar nome ao container, -e= variaveis de ambiente que podemos passar de acordo com a imagem utilizada, nesse caso setados que o mysql vai gerar uma senha aleatoria para o usuário root, que podemos pegar depois com o comando docker container log img, mysql= imagem do container), caso o container ja exista esse comando da erro, e basta executar: docker run -d -e MYSQL_RANDOM_ROOT_PASSWORD=true mysql
